# AeroSky Workspace Rules

## The AeroSky Editorial Standard

All content on the AeroSky platform must adhere to the following core guidelines:

1. **Tone and Voice:** Engineering-first, community-driven, and technically accurate. Avoid marketing fluff, buzzwords, and passive invitations. Draw inspiration from platforms like Cloudflare (engineering credibility), GitHub (community collaboration), and FlightRadar24 (aviation expertise).
2. **Selective "Sovereign" Usage:** Do not overuse "sovereign." Use "community-powered airspace intelligence for India" or "independent aviation intelligence platform" for public-facing pages. Reserve "sovereign airspace intelligence infrastructure" for select B2B, About, and technical document pages.
3. **No Overstatement:** Frame all copy around *building* and *creating* the network. Never claim that AeroSky already operates a national tracking grid.
4. **Identity First:** Keep the "Become an AeroCaptain" identity as the primary call to action, followed by explaining the task ("Host a ground station").
5. **Standard Terminology:** Always use:
   - AeroCaptain / AeroCaptains
   - India's Airspace Intelligence Network
   - Community
   - Coverage
   - Insights

### The Five-Question Page Check
Every page must answer these five questions within the first few seconds of a user visiting:
1. **What is AeroSky?**
2. **Why does it matter?**
3. **Why should I trust it?**
4. **How can I contribute or benefit?**
5. **What should I do next?**
